Data Sources
=====

Here is a bunch of data sources, and eventually code for how do download them & analyze them.

 * OpTC - a 2019 project from DARPA to record a Red-team exercise. 
 Includes json records from the hosts, and Bro logs of the network traffic
   * [Data](https://github.com/FiveDirections/OpTC-data) - The github repo for the data itself
 * Publicly available pcap - A collection of Packet captures from various places (exercises, malware dumps, etc)
   * [Data](https://www.netresec.com/index.ashx?page=PcapFiles)
 * Detecting Malicious URLs - a Paper from UCSD on detecting malicious urls
   * [Data](https://www.sysnet.ucsd.edu/projects/url/)
 * Los Alamos National Labs collection - A set of logs from systems and network for a 58-day period.
   * [Data](https://csr.lanl.gov/data/)
 * Malware Capture Facility - Network traffic including malware caputured from teh CTU university in Czecia
   * [Data](https://mcfp.weebly.com/mcfp-dataset.html)
 * Cisco Ie - Anomaly detection datasets released by Cisco
   * [Data](https://github.com/cisco-ie/telemetry)
 * Canadian Institute for Cybersecurity datasets - Collection of datasets for things like DDos, VPN vs Non-VPN traffic, etc.
   * [Data](https://www.unb.ca/cic/datasets/index.html)
 * University California Irvine spambase - a collection of emails flagged as spam.
   * [Data](https://archive.ics.uci.edu/ml/datasets/spambase)
 * SMS spam Kaggle competition - a Kaggle Machine learning competition to identify SMS spam
   * [Data](https://www.kaggle.com/uciml/sms-spam-collection-dataset)
 * Stratosphere datasets - a collection of network traffic (both benign and malicious). Note: malicious comes from MCFP 
 listed above.
   * [Data](https://www.stratosphereips.org/datasets-overview)
 * UNSW Canberra cyber datasets 
   * [Data](https://www.unsw.adfa.edu.au/unsw-canberra-cyber/cybersecurity/ADFA-IDS-Datasets/)
